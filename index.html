About me

[카카오 플랫폼 경험]

카카오 공동체는 카카오에서 제공하는 인프라를 사용합니다. 소스코드의 형상관리와 서비스 CI/CD 에 대해 갖춰져 있으며, 카카오의 서비스가 아닌 공동체 서비스의 경우, 특성을 고려하여 VM을 별도로 설치하여 서비스를 구축할 수 있습니다. 현 회사에 입사하여, T-Fac 서비스에 대해 유지보수하면서, 카카오의 클라우드 서비스를 이용해 버그 및 요구사항 반영과 쿠버네티스 환경에서의 배포가 이루어지는 카카오 플랫폼을 경험하였으며, AWS의 인스턴스와 같은 VM 설치를 하고, 서비스를 배포하면서, 내부 보안망으로 인해 외부인터페이스 연동이 안되는 부분을 분석하며, 내부 협업툴을 통해 보안설정을 진행하여 서비스가 안정적으로 서빙될 수 있게 하였습니다. 카카오 공동체인 만큼, 직접적으로 인프라에 대한 지원을 사내에서 지원받을 수 없어, 협업툴을 이용해 주도적으로 요청하고, 확인이 필요했는데, 물어보는 것에 대한 망설임이 없는 성향이 업무를 함에 있어 큰 도움이 되었습니다.

[다양한 운영환경에서 성공적인 운영/유지보수 서비스 제공]

on-promise 로 제공된 카카오톡 챗봇 스킬서버 및 Skillbuilder 서비스를 운영/유지보수하였습니다. 고객사마다 다양한 클라우드 서비스 및 자체 서버를 사용하고 있어, 고객사별 VPN내용과 서버 관련한 정보들을 문서에 작성하였습니다. 또한, 고객사별 요구사항을 위해 직접 커뮤니케이션하여 일정과 업무를 작성하고, 카카오 장애에 대한 대응 시에도 장애 템플릿 문서를 작성하여 배포하였습니다. 특히, 장애대응을 위해, 서버구성도를 파악하여, WEB -> WAS로 프록시를 연결하는 구성에서 WEB 서버의 nginx 에 대한 configure 설정을 다루어 이슈를 해결한 경험이 있습니다. 고객의 요구사항을 일목요연하게 정리하여, 회사에서 대응할 수 있는, 또는 대응해야하는 범위인지를 파악하고, 중간에서 조율하면서 고객사에게 만족과 신뢰를 안겨줌으로써 회사의 신뢰도를 높이는데 기여하였습니다.

[남다른 스킬셋]

C/C++/Java를 바탕으로 임베디드 소프트웨어를 개발을 했고, 백앤드 개발자로 전향하여 업무를 진행하였습니다. 임베디드 소프트웨어 개발자로서, 펌웨어 및 임베디드 리눅스가 설치된 작은 모듈에서 작업하면서, 통신의 원리와 네트워크 지식등을 쌓았고, 리눅스 CLI 에 대한 사용을 익혔습니다. 이러한 경험이, 백앤드 개발자로서 업무를 진행하면서, VM에서 인프라를 다룰 때 굉장히 큰도움이 되었으며, 특히, 제한된 리소스를 제공하는 VM에서 서비스의 에러를 파악하고, 보안망에 구축된 인프라들에 대해 네트워크 이슈가 발생했을 때의 원인을 찾는데 도움이 되었습니다. 백앤드 개발자로 전향하면서, 스프링부트 3 기반의 REST API 서비스와 Thymeleaf와JQuery를 이용한 웹서비스를 구축하였습니다. 특히, log4j2 에 대한 설정과 스프링 클라우드를 이용한 게이트웨이 설정, 프로필 그룹화 설정, Open Feign, XSS Filter 등의 설정에 대해 문서를 작성하여 공유하여, 서비스 구축에 이바지하였습니다. REST API 서비스를 개발하면서, 서비스에 대한 도메인과 ERD 설계를 비롯하여, 효율적인 개발을 위해 Redis를 사용한 캐싱전략을 사용, 데이터 정합성과 동시성이슈를 해결하기 위해 Lock 전략을 사용하였습니다. 또한, 카카오톡 메세징 서비스에 대한 전반적인 내용에 대해 이해하고 있으며, 카카오톡 채널과 챗봇, 레거시 API와 연동을 위한 챗봇 응답 서버 구축에 대한 이해를 갖고, 다양한 고객사의 챗봇 응답 서버를 구축/유지보수 하였습니다. 이를 바탕으로, 고객에게 제공되는 서비스의 API 개발을 할 수 있으며, 카카오와의 협업이 있다면, 해당 업무에 대해 주도적으로 이끌어나갈 수 있습니다.
